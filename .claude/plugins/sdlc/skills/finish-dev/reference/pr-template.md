# PR Body Template

Reference template for finish-dev PR bodies. Claude populates each section with real content — do not use placeholder text.

## Structure

```markdown
## Summary

<1-2 sentence overview of what this branch delivers as an aggregate>

### Child Issues

| Issue | Title |
|-------|-------|
| #N | <title> |
| #M | <title> |

## Test Plan

<Context-aware verification plan — NOT a generic checklist. Generated by analyzing:
- The PR diff (git diff <target>...HEAD)
- The parent issue description and acceptance criteria
- Child issue titles and descriptions>

- [ ] <specific test step 1>
- [ ] <specific test step 2>
- [ ] ...

---

Closes #<ISSUE_NUM>
```

## Guidance for Populating

### Summary
Describe the aggregate outcome — what does merging this branch deliver? Do not list individual child PRs. Focus on the capability or milestone this completes.

### Child Issues
List every child issue from the parent's checklist section (`## Stories` for large features, `## Features` for epics, `## Epics` for PIs). Include issue number and title for each.

### Test Plan
Read the actual diff and issue context. Include:
- **Functional verification** — steps tied to specific acceptance criteria from the issue
- **Integration checks** — do the pieces work together as a whole?
- **Regression considerations** — what could this break in adjacent areas?
- **Edge cases** — anything surfaced by reviewing the diff

Each item should reference specific functionality or files changed. Avoid generic steps like "verify it works."

### Closes Footer
Always include `Closes #N` — GitHub auto-closes the issue when the PR is merged.
